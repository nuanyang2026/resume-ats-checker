## 1. 产品概述

Resume ATS Checker 是一款帮助求职者检测简历是否符合 ATS（Applicant Tracking System，申请人跟踪系统）筛选标准的在线工具。用户上传简历后，系统通过 AI 分析给出评分、关键词匹配情况和改进建议，帮助用户提升简历的过筛率。

**核心价值**
- 即时分析，无需注册
- 无数据存储，保护用户隐私
- AI 驱动，分析结果精准实用

---

## 2. 目标用户

- 正在求职的应届生 / 职场人士
- 海外求职者（英文简历优先）
- 希望优化简历通过率的候选人

## 3. 功能需求

### 3.1 简历上传

| 功能点 | 说明 |
| --- | --- |
| 支持格式 | PDF、Word（.doc / .docx） |
| 文件大小限制 | 最大 5MB |
| 上传方式 | 点击上传 / 拖拽上传 |
| 上传反馈 | 上传进度提示、成功/失败状态 |

### 3.2 职位描述输入（可选）

用户可粘贴目标职位的 JD（Job Description），系统据此做针对性关键词匹配分析。若不填写，则做通用 ATS 分析。

### 3.3 AI 分析

调用 Claude API 对简历进行分析，返回以下维度结果：

**① ATS 总体评分（0–100 分）**
- 评分维度：格式规范性、关键词密度、结构完整性、可读性
- 显示评分 + 等级标签（优秀 / 良好 / 待改进）

**② 关键词分析**
- 从简历中提取高频关键词
- 若用户填写了 JD，对比 JD 关键词，展示命中率
- 标注缺失的高价值关键词

**③ 格式检测**
- 是否使用了 ATS 不友好的格式（表格、图片、多栏布局等）
- 字体、字号是否规范

**④ 改进建议**
- 针对每个扣分项给出具体改进建议
- 建议用简洁的 Action Item 形式呈现（最多 5–8 条）

### 3.4 结果展示

- 评分卡：大字展示总分 + 环形进度图
- 关键词标签云：命中 / 缺失用颜色区分
- 改进清单：可折叠展开的条目列表
- 支持复制分析报告文本

## 4. 技术方案

### 4.1 整体架构

```
用户浏览器（前端 Next.js 14 App Router）
    │
    ├─ 上传简历（PDF/Word）
    ├─ 前端解析文档内容（pdf.js / mammoth.js）
    └─ 请求 Cloudflare Workers API Route
              │
              ├─ 内存处理简历文本（不落盘，不存储）
              ├─ 调用 Claude API 分析
              └─ 返回分析结果（JSON）
                        │
                        └─ 前端渲染展示
```

> ⚠️ 简历内容全程不落盘、不存储，仅在 Workers 内存中临时处理。

### 4.2 技术栈

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端框架 | Next.js 14（App Router） | SSR + 静态生成 |
| 部署平台 | Cloudflare Pages | 全球 CDN，免费额度充足 |
| 后端 | Cloudflare Workers | API Route 处理，边缘计算 |
| AI 能力 | Claude API（Anthropic） | 简历分析核心能力 |
| 文档解析 | pdf.js / mammoth.js | 浏览器端 PDF & Word 解析 |
| UI 组件 | Tailwind CSS + shadcn/ui | 快速构建界面 |
| 支付（后期） | Stripe | 付费功能接入 |

### 4.3 数据流

1. 用户上传简历 → 浏览器端解析为纯文本
2. （可选）用户粘贴 JD
3. 前端将文本发送至 Cloudflare Workers API Route
4. Workers 在内存中处理，拼接 Prompt，调用 Claude API
5. 解析返回 JSON，响应给前端
6. 前端渲染分析结果
7. 请求结束后，Workers 内存自动释放，无任何持久化存储

### 4.4 Prompt 设计

```
You are a professional resume ATS (Applicant Tracking System) expert.

Analyze the following resume and return a structured JSON result:

Resume Content:
<resume_text>

{JD if provided:}
Job Description:
<jd_text>

Return JSON with this structure:
{
  "ats_score": 0-100,
  "score_level": "Excellent|Good|Needs Improvement",
  "keywords_found": ["keyword1", "keyword2"],
  "keywords_missing": ["keyword3", "keyword4"],
  "format_issues": ["issue1", "issue2"],
  "improvement_suggestions": ["suggestion1", "suggestion2"],
  "summary": "One paragraph summary"
}
```

### 4.5 Cloudflare Workers API Route 示例

```javascript
// workers/api/analyze.js
export default {
  async fetch(request, env) {
    if (request.method !== 'POST') {
      return new Response('Method Not Allowed', { status: 405 });
    }

    const { resumeText, jdText } = await request.json();

    // 内存处理，构建 Prompt
    const prompt = buildPrompt(resumeText, jdText);

    // 调用 Claude API
    const result = await callClaude(prompt, env.ANTHROPIC_API_KEY);

    // 返回结果，不做任何存储
    return new Response(JSON.stringify(result), {
      headers: { 'Content-Type': 'application/json' },
    });
  },
};
```

## 7. MVP 范围（第一版）

**✅ 包含**

## 8. 风险与注意事项

| 风险 | 说明 | 应对 |
| --- | --- | --- |
| Cloudflare Workers 限制 | 免费版 CPU 时间 10ms/请求，调用 AI 可能超时 | 升级到 Workers Paid（$5/月）或用 Workers Unbound |
| PDF 解析失败 | 扫描版 PDF 无法提取文本 | 给出友好提示，建议上传文字版 PDF |
| API 费用 | 单次调用约 1000–3000 Token | 成本可控，后期可加限流或付费墙 |
| CORS 配置 | Workers 需正确配置跨域头 | 在 Workers 中统一处理 CORS |
| 响应超时 | AI 响应可能超 15 秒 | 添加 loading 动画 + 超时重试机制 |
